Session 5

Advanced Query Patterns & Optimization

Variable-Length Path Patterns

  • Basic Syntax: Use * notation for variable paths
  • Depth Control: Specify minimum and maximum hops
  • Relationship Filtering: Filter by type and properties
  • Performance Considerations: Limit depth to avoid explosion
Variable Path Examples:
(a)-[:KNOWS*1..3]->(b) // 1 to 3 hops
(a)-[:KNOWS*]->(b) // Unlimited depth
(a)-[:KNOWS|FOLLOWS*2..4]->(b) // Multiple types

Complex Pattern Matching

  • Multiple Patterns: Combine different relationship paths
  • Optional Patterns: Use OPTIONAL MATCH for flexible queries
  • Pattern Comprehensions: Collect paths and relationships
  • Conditional Logic: Apply WHERE within patterns
Complex Pattern Example:
Find users who are friends and work colleagues:
MATCH (a:User)-[:FRIENDS]->(b:User), (a)-[:COLLEAGUES]->(b)

Subqueries and WITH Clauses

  • WITH Clause: Chain query parts and transform data
  • Aggregation Pipelines: Build complex analytical flows
  • Filtering Results: Apply conditions on aggregated data
  • Ordering and Limiting: Control result sets between steps
Subquery Pattern:
MATCH (u:User)-[:POSTED]->(p:Post)
WITH u, count(p) AS posts
WHERE posts > 10
RETURN u.name, posts

Query Performance Optimization

Best Practices

  • Use labels to narrow searches
  • Index frequently queried properties
  • Limit variable-length paths
  • Use EXPLAIN and PROFILE

Common Pitfalls

  • Cartesian products without filters
  • Unlimited path traversals
  • Missing indexes on WHERE clauses
  • Large result sets without LIMIT

Index Usage and Query Planning

  • Index Types: Range, text, point, and composite indexes
  • Query Planner: How Cypher chooses execution paths
  • Hints: Force specific index usage when needed
  • Statistics: Database statistics influence planning
Performance Tip: Use PROFILE to see actual execution times and row counts, not just plans.

Memory Management

  • Eager Operations: Operations that load all data into memory
  • Lazy Evaluation: Stream processing when possible
  • Memory Configuration: Heap and page cache settings
  • Large Query Handling: Strategies for big datasets

Memory Optimization Strategy

Use LIMIT early, avoid DISTINCT on large sets, and consider periodic COMMIT for write operations

Aggregations & Statistical Analysis

Core Aggregation Functions

  • COUNT: Count nodes, relationships, or patterns
  • SUM, AVG: Mathematical aggregations on numeric properties
  • MIN, MAX: Find extremes in datasets
  • COLLECT: Gather values into lists
  • DISTINCT: Remove duplicates from aggregations
Aggregation Examples:
count(DISTINCT u) // Unique users
avg(u.age) // Average age
collect(u.name) // List of names

Grouping Operations

  • Implicit Grouping: Non-aggregated fields create groups
  • Multiple Grouping: Group by multiple properties
  • Hierarchical Grouping: Nested grouping patterns
  • Conditional Grouping: Group based on calculated fields
Grouping Example:
MATCH (u:User)-[:LIVES_IN]->(c:City)
RETURN c.name, count(u) AS residents
ORDER BY residents DESC

Temporal Aggregations

  • Date Grouping: Group by year, month, day
  • Time Windows: Rolling aggregations over periods
  • Duration Calculations: Time differences and trends
  • Temporal Filtering: Recent data and time ranges
Temporal Example:
MATCH (u:User)-[:POSTED]->(p:Post)
WHERE p.timestamp > datetime() - duration('P30D')
RETURN date.truncate('month', p.timestamp) AS month,
       count(p) AS posts

Statistical Analysis Functions

Descriptive Statistics

percentileDisc(), percentileCont(), stDev(), variance()

List Operations

size(), head(), tail(), range(), reduce()

Mathematical Functions

abs(), round(), sqrt(), log(), exp()

String Analytics

size(), split(), replace(), substring()

Advanced Analytics Patterns

  • Percentile Analysis: Distribution analysis and outliers

  • Cohort Analysis: User behavior over time segments
Analytics Pattern: Use COLLECT to gather time series data, then apply mathematical functions for trend analysis.

Session 7

Graph Algorithms Fundamentals

Pathfinding Algorithms

Shortest Path

Find minimum distance between nodes using BFS or Dijkstra

All Shortest Paths

Find all paths with minimum distance between nodes

Weighted Shortest Path

Consider relationship weights for optimal routing

A* Algorithm

Heuristic pathfinding with estimated costs

Pathfinding Implementation in Cypher

Basic Shortest Path:
MATCH path = shortestPath((a:User {name: 'Alice'})-[:KNOWS*]-(b:User {name: 'Bob'}))
RETURN path, length(path)

All Shortest Paths:
MATCH paths = allShortestPaths((a)-[:KNOWS*]-(b))
RETURN paths

Weighted Path:
MATCH path = (a)-[:CONNECTED*]-(b)
RETURN path, reduce(weight = 0, r IN relationships(path) | weight + r.cost) AS total_cost

Centrality Algorithms

  • Degree Centrality: Number of direct connections
  • Betweenness Centrality: How often node appears on shortest paths
  • Closeness Centrality: Average distance to all other nodes
  • PageRank: Importance based on incoming links
  • Eigenvector Centrality: Importance of connected nodes
Use Cases: Identify influencers, key connectors, and important nodes in networks.

Centrality Calculation Methods

Native Cypher

  • Simple degree calculations
  • Basic betweenness approximations
  • Custom scoring algorithms
  • Good for small networks

Graph Data Science Library

  • Optimized algorithm implementations
  • Handles large-scale networks
  • Parallel processing capabilities
  • Enterprise-grade performance

Community Detection Algorithms

  • Louvain Method: Modularity-based community detection
  • Label Propagation: Fast community discovery
  • Weakly Connected Components: Find disconnected subgraphs
  • Strongly Connected Components: Directed graph communities
  • Triangle Counting: Measure clustering coefficient

Community Detection Benefits

Identify natural groups, optimize marketing, and understand network structure

Algorithm Complexity and Performance

Algorithm Time Complexity Space Complexity Best Use Case
BFS Shortest Path O(V + E) O(V) Unweighted paths
Dijkstra O((V + E) log V) O(V) Weighted paths
PageRank O(V + E) per iteration O(V) Influence ranking
Louvain O(V log V) O(V + E) Community detection

Choosing the Right Algorithm

Network Analysis Goals

  • Pathfinding: Routing and navigation
  • Influence: Marketing and recommendations
  • Communities: Segmentation and clustering
  • Connectivity: Robustness and resilience

Performance Considerations

  • Graph Size: Number of nodes and edges
  • Algorithm Complexity: Time and space requirements
  • Update Frequency: Static vs dynamic networks
  • Accuracy Needs: Exact vs approximate results
