Session 5: Advanced Query Patterns & Optimization

Building Complex Graph Traversals

Variable-Length Path Fundamentals

Variable-length paths allow traversing relationships of unknown or varying depth, essential for exploring complex network structures.

Core Concepts

  • Syntax: (start)-[:RELATIONSHIP*min..max]->(end)
  • Depth Control: Specify minimum and maximum traversal depths
  • Bidirectional: Use -[:REL*]- for undirected traversal
  • Multiple Types: [:FOLLOWS|LIKES*] for multiple relationship types
  • Property Access: Extract node/relationship properties from paths

Variable-Length Path Examples

Basic Friend-of-Friend Discovery:
MATCH (alice:User {name: 'Alice'})-[:FOLLOWS*1..2]->(suggestions:User)
WHERE alice <> suggestions
RETURN DISTINCT suggestions.name
Path Property Filtering:
MATCH path = (start)-[:FOLLOWS*2..4]->(end)
WHERE ALL(node IN nodes(path)[1..-1] WHERE node.active = true)
AND ALL(rel IN relationships(path) WHERE rel.strength > 0.5)
RETURN path
Critical: Always set maximum depth to prevent infinite traversals that can crash your database

Variable-Length Path Applications

Real-World Use Cases

  • Social Networks: Find friends-of-friends, influence propagation chains
  • Recommendation Systems: "People you may know" based on mutual connections
  • Supply Chains: Trace products through multi-tier supplier networks
  • Fraud Detection: Identify suspicious transaction chains
  • Knowledge Graphs: Navigate semantic relationships of varying depth
  • Network Analysis: Calculate reachability and connectivity metrics

Complex Pattern Matching

Advanced pattern matching combines multiple constraints and conditions to create sophisticated graph queries.

Key Techniques

  • Multiple Relationships: [:FOLLOWS|LIKES|SHARES*] traverse different connection types
  • Conditional Filtering: ALL() and ANY() functions for path validation
  • Path Functions: nodes(path), relationships(path), length(path)
  • Strategic WITH: Control query execution flow and intermediate results
Complex Example:
MATCH path = (user)-[:FOLLOWS*1..3]->(target)
WHERE ANY(middle IN nodes(path)[1..-1] WHERE middle.verified = true)
WITH path, target ORDER BY length(path) LIMIT 10
RETURN target.name, length(path) AS hops

Query Optimization Strategies

Performance optimization is critical for production graph applications, especially with complex patterns.

  • Index Usage: Create indexes on frequently filtered properties
  • Query Planning: Use EXPLAIN to understand execution plans
  • Memory Management: Monitor heap usage for large traversals
  • Early Filtering: Apply WHERE clauses as early as possible
  • Result Limiting: Use LIMIT strategically throughout query pipeline
  • Profiling: PROFILE queries to identify bottlenecks
Best Practice: Start with specific nodes using indexed properties, then expand outward

Session 6: Aggregations & Statistical Analysis

Business Intelligence with Graph Data

Advanced Aggregation Functions

Graph aggregations transform raw relationship data into meaningful business insights and statistical summaries.

Core Aggregation Functions

  • COUNT: count(nodes), count(relationships), count(DISTINCT property)
  • Mathematical: sum(), avg(), min(), max(), stdev()
  • COLLECT: collect(property) - aggregate values into lists
  • Percentiles: percentileCont(), percentileDisc() for distribution analysis
  • Conditional: count(CASE WHEN condition THEN 1 END)
Example - User Engagement Analysis:
MATCH (u:User)-[:POSTED]->(p:Post)<-[:LIKES]-(liker)
RETURN u.name, count(p) AS posts, count(liker) AS total_likes,
       avg(count(liker)) AS avg_likes_per_post

Session 7: Graph Algorithms Fundamentals

Algorithmic Network Analysis

Pathfinding Algorithms

Pathfinding algorithms solve routing and connection problems in graph networks, essential for recommendations and optimization.

Core Algorithm Types

Shortest Path: shortestPath(), allShortestPaths() - find minimal connection routes
Dijkstra: Weighted shortest paths considering relationship costs/distances
Breadth-First Search (BFS): Explore neighbors level by level, unweighted paths
Depth-First Search (DFS): Deep exploration before backtracking
Weighted Shortest Path Example:
MATCH (start:Location {name: 'New York'}), (end:Location {name: 'Los Angeles'})
CALL gds.shortestPath.dijkstra.stream('transportation', {
  sourceNode: start, targetNode: end, relationshipWeightProperty: 'distance'
}) YIELD path RETURN path

Pathfinding Business Applications

Real-world pathfinding powers recommendation engines, logistics optimization, and network analysis.

  • Social Recommendations: "People you may know" via shortest social paths
  • Supply Chain Optimization: Find most efficient routes through supplier networks
  • Influence Propagation: Model how information/trends spread through networks
  • Fraud Detection: Identify suspicious transaction chains and money flows
  • Network Efficiency: Optimize connection strategies and reduce bottlenecks
  • Content Discovery: Route users to relevant content through interest graphs
Algorithm Selection: Use BFS for unweighted/social networks, Dijkstra for logistics/cost optimization, A* for geographic routing

Centrality Measures

Centrality algorithms identify the most important, influential, or strategic nodes in a network.

Key Centrality Types & Calculations

Degree Centrality: count((node)-[:REL]-(other)) - direct connection count
Betweenness Centrality: Measures how often a node lies on shortest paths between others
Closeness Centrality: Average distance to all other reachable nodes
PageRank: Iterative importance based on connections from important nodes
Eigenvector Centrality: Importance based on importance of neighbors
Degree Centrality Example:
MATCH (user:User)
RETURN user.name, size((user)-[:FOLLOWS]-()) AS degree_centrality
ORDER BY degree_centrality DESC LIMIT 10

Centrality Strategic Applications

Centrality insights drive marketing strategies, risk management, and network optimization decisions.

  • Influencer Marketing: High PageRank/degree users for maximum campaign reach
  • Risk Management: High betweenness nodes are critical failure points
  • Target Marketing: Closeness centrality identifies users who can reach entire network quickly
  • Network Resilience: Protect high-centrality nodes from attacks or failures
  • Information Bottlenecks: Betweenness centrality reveals communication chokepoints
  • Community Leaders: Local centrality measures identify micro-influencers
Strategic Value: Centrality analysis can identify key employees, critical infrastructure, or high-value customers for targeted strategies

Community Detection Algorithms

Community detection identifies clusters and groups within networks, revealing hidden organizational structures.

Detection Methods & Approaches:
• Attribute-Based: Group by location, profession, interests using node properties
• Structural: Analyze connection patterns using modularity and clustering
• Behavioral: Cluster based on activity patterns and interaction frequency
• Hybrid: Combine multiple signals for robust community identification
• Hierarchical: Detect communities at multiple scales and resolutions
Simple Community Detection:
MATCH (u:User) WHERE u.location IS NOT NULL
SET u.community = 'LOC_' + replace(u.location, ' ', '_')
RETURN u.community, count(u) AS size ORDER BY size DESC
